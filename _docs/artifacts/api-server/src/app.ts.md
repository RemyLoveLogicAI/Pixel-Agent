## Overview

This module imports Express (including the Express type), a CORS middleware, the project's API router, and two error-handling middlewares. It instantiates an Express app, registers cors(), express.json(), and express.urlencoded({ extended: true }), mounts the imported router at "/api", then attaches notFound and errorHandler middleware.

The file contains only top-level application configuration intended to be imported by a separate server bootstrapper that starts the HTTP listener. Route and error handling implementations live in the referenced internal modules.

## Architecture Notes

- Top-level Express app configuration pattern: the module creates and configures a single app instance and exports it for a separate server bootstrapper to start listening.
- Separation of concerns: routing is delegated to an internal router module mounted at '/api', and error-handling concerns are delegated to dedicated middleware in ./middlewares/error-handler.js.
- Middleware order matters: CORS and body parsers are applied before routes; the 404 and error handler middleware are registered after routes so they can correctly handle unmatched routes and errors.

## Usage Examples

### Main use case

Import the default-exported app in the server entrypoint (e.g., import app from './app') and start the server (e.g., app.listen or create an HTTP server). The app will already have CORS, JSON/urlencoded body parsing, the '/api' router mounted, and the 404/error handlers attached.

## Maintenance Notes

- Keep middleware registration order: body parsers and cors must be registered before mounting the router; notFound and errorHandler must be registered after the router.
- The file imports internal modules with explicit '.js' extensions (./routes/index.js and ./middlewares/error-handler.js) — ensure build/transpile output or runtime supports ESM resolution with .js paths.
- Ensure the exported router and error-handler middleware signatures match Express middleware expectations (req, res, next) since this file relies on them being compatible when mounted.
